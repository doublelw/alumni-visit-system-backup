# 校友审核功能增强说明

## 功能概述

校友入校登记系统的管理后台校友审核功能已经全面增强，支持多状态管理、历史记录查看、高级筛选和批量操作。

## 新增功能特性

### 1. 分类状态管理 ✨

#### 📊 统计卡片
- **待审核** (橙色): 实时显示待审核校友数量
- **已通过** (绿色): 显示已审核通过的校友数量
- **已拒绝** (红色): 显示已审核拒绝的校友数量
- **全部校友** (蓝色): 显示所有校友总数

点击统计卡片可快速切换到对应状态列表。

#### 🏷️ 状态筛选标签
- **待审核**: 默认显示，需要管理员审核的新申请
- **已通过**: 已通过审核的校友档案
- **已拒绝**: 已被拒绝的校友档案
- **全部**: 查看所有状态下的校友

### 2. 高级搜索和筛选 🔍

#### 🔎 搜索功能
- 支持按校友姓名搜索
- 支持按学号搜索
- 支持按班级名称搜索
- 实时搜索，无需点击按钮

#### 🎯 多维度筛选
- **院系筛选**: 按毕业院系分类查看
- **年级筛选**: 按毕业年份分类查看
- **组合筛选**: 可同时使用多个筛选条件
- **一键重置**: 快速清除所有筛选条件

### 3. 详细信息表格 📋

#### 表格列信息
- **校友信息**: 姓名、手机号、邮箱（复合显示）
- **学号**: 校友学号
- **班级**: 毕业班级
- **院系**: 毕业院系
- **专业**: 专业信息
- **毕业年份**: 毕业时间
- **申请时间**: 提交审核的日期
- **审核状态**: 当前审核状态（彩色徽章）
- **审核人**: 执行审核的管理员姓名
- **操作按钮**: 针对不同状态的操作按钮

### 4. 智能操作按钮 🎛️

#### 待审核状态
- ✅ **通过审核**: 批准校友申请
- ❌ **拒绝审核**: 拒绝校友申请

#### 已通过状态
- 👁️ **查看详情**: 查看校友详细信息和审核记录

#### 已拒绝状态
- 👁️ **查看详情**: 查看拒绝原因和相关记录
- 🔄 **重新审核**: 将校友重新提交到待审核队列

### 5. 审核历史记录 📚

#### 自动记录
- **审核时间**: 记录审核操作的具体时间
- **审核人员**: 显示执行审核的管理员姓名
- **审核结果**: 通过/拒绝状态一目了然
- **审核备注**: 可选的审核说明和备注

### 6. 分页和排序 📖

#### 智能排序
- 待审核校友优先显示
- 按申请时间倒序排列
- 确保最新申请得到优先处理

#### 高效分页
- 每页显示20条记录
- 支持快速页面跳转
- 显示总数和当前页信息

### 7. 响应式设计 📱

#### 多端适配
- **桌面端**: 完整功能体验
- **平板端**: 自适应布局调整
- **手机端**: 优化的移动界面

#### 交互优化

## API接口说明

### 获取校友列表
```http
GET /api/admin/alumni-approve?page=1&per_page=20&status=pending&search=张三&department=计算机学院&graduation_year=2020
```

#### 参数说明
- `page`: 页码 (默认: 1)
- `per_page`: 每页数量 (默认: 20)
- `status`: 状态筛选 (pending/approved/rejected/all)
- `search`: 搜索关键词
- `department`: 院系筛选
- `graduation_year`: 年级筛选

#### 返回数据
```json
{
  "alumni": [...],           // 校友列表
  "statistics": {           // 统计信息
    "pending": 0,
    "approved": 0,
    "rejected": 0,
    "total": 0
  },
  "filters": {              // 筛选选项
    "departments": [...],
    "graduation_years": [...]
  },
  "pagination": {           // 分页信息
    "page": 1,
    "pages": 0,
    "total": 0,
    "has_next": false,
    "has_prev": false
  }
}
```

### 审核操作
```http
POST /api/admin/alumni/{profile_id}/approve
```

#### 请求体
```json
{
  "approve": true,    // true: 通过, false: 拒绝
  "note": "审核备注"   // 可选的审核说明
}
```

### 重新审核
```http
POST /api/admin/alumni/{profile_id}/reapprove
```

## 管理流程

### 标准审核流程
1. **查看待审核**: 进入校友审核页面，默认显示待审核列表
2. **查看详情**: 点击校友姓名查看详细信息
3. **执行审核**: 根据校友资料选择通过或拒绝
4. **记录完成**: 系统自动记录审核操作和审核人

### 特殊场景处理
1. **误判处理**: 对已拒绝的校友可点击"重新审核"
2. **信息查询**: 使用搜索和筛选功能快速定位校友
3. **状态统计**: 通过统计卡片快速了解各状态数量
