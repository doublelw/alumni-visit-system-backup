# 校友入校登记系统 - 自动化测试

## 📋 概述

这个自动化测试系统可以完整模拟校友入校登记的整个流程，从访客申请到保安验证通过的全过程自动化操作。

## 🚀 快速开始

### 1. 环境准备

确保以下服务正在运行：

- **后端服务**: `http://localhost:5000`
- **前端服务**: `http://localhost:8080`
- **数据库**: SQLite数据库已初始化

### 2. 启动服务

```bash
# 启动后端服务
cd backend
python -m flask run --host=0.0.0.0 --port=5000

# 启动前端服务（新终端）
cd frontend
python -m http.server 8080
```

### 3. 运行自动化测试

1. 打开浏览器访问 `http://localhost:8080/test_runner.html`
2. 确保浏览器允许弹窗（用于打开多个测试窗口）
3. 点击"🚀 开始自动化测试"按钮
4. 测试将自动执行完整流程

## 🎯 测试流程

### 访客端测试
1. **用户注册**: 自动生成测试用户账户
2. **用户登录**: 使用注册的账户登录
3. **提交访问申请**: 填写访问信息并提交
4. **人脸信息录入**: 录入人脸照片用于验证

### 教师端测试
1. **教师登录**: 使用预设教师账户登录
2. **查看待审批申请**: 查看访客提交的申请
3. **审批访问申请**: 批准访客的访问请求
4. **生成访问二维码**: 为已批准的申请生成二维码

### 保安端测试
1. **保安登录**: 使用预设保安账户登录
2. **二维码验证**: 扫描或手动输入二维码验证访客
3. **人脸识别验证**: 进行人脸识别比对
4. **访客放行**: 完成验证后允许访客进入

## 🛠️ 自定义配置

### 修改测试数据

在 `test_automation.js` 中的 `testData` 对象可以修改测试参数：

```javascript
testData: {
    visitor: {
        username: 'test_visitor_' + Date.now(),
        password: 'Test123456',
        realName: '测试访客',
        // ... 其他参数
    },
    teacher: {
        username: 'teacher001',
        password: 'teacher123'
    },
    security: {
        username: 'security001',
        password: 'security123'
    }
}
```

### 调整测试步骤

可以在 `steps` 数组中修改测试步骤顺序或添加新步骤：

```javascript
steps: [
    'openAllWindows',
    'visitorRegister',
    'visitorLogin',
    // ... 自定义步骤
]
```

## 🔍 故障排除

### 常见问题

1. **弹窗被阻止**: 确保浏览器允许该网站的弹窗
2. **服务连接失败**: 检查后端和前端服务是否正常运行
3. **元素定位失败**: 页面结构发生变化，需要更新选择器
4. **权限不足**: 确保测试账户具有相应权限

### 调试模式

在浏览器开发者工具中查看详细日志：

```javascript
// 在控制台输出详细测试信息
console.log('测试步骤详情:', automation.currentStep);
console.log('测试数据:', automation.testData);
```

## 🚨 注意事项

1. **浏览器兼容性**: 建议使用 Chrome 或 Edge 浏览器
2. **网络环境**: 确保本地网络连接正常
3. **系统资源**: 测试会打开多个窗口，确保系统资源充足
4. **数据清理**: 测试数据会保存在数据库中，定期清理测试数据
5. **安全性**: 仅在开发环境中使用，不要在生产环境运行
