## 📊 测试执行计划

### 阶段1：冒烟测试 (30分钟)
**目标**: 验证核心功能可正常工作

1. 启动服务器
2. 测试登录功能（3种角色）
3. 测试通行码生成
4. 测试访客登记
5. 测试基础流程

**脚本**: `test_smoke.py`

---

### 阶段2：功能测试 (2小时)
**目标**: 验证所有功能点正常工作

执行所有测试模块：
```bash
# 单元测试
pytest tests/unit/ -v

# 集成测试
pytest tests/integration/ -v

# API测试
pytest tests/api/ -v
```

---

### 阶段3：端到端测试 (1小时)
**目标**: 验证完整用户流程

使用Playwright进行E2E测试：
```bash
pytest tests/e2e/ -v --headed
```

**测试场景**:
1. 家长注册 → 生成通行码
2. 教师审批访客
3. 门卫验证访客
4. 学生请假流程
5. 统计数据查看

---

### 阶段4：性能测试 (1小时)
**目标**: 验证系统性能指标

**性能指标**:
- 响应时间 < 2秒 (95th percentile)
- 并发用户 > 100
- 数据库查询 < 500ms
- API吞吐量 > 100 req/s

**工具**: Apache JMeter / Locust

---

### 阶段5：安全测试 (30分钟)
**目标**: 发现并修复安全漏洞

**检查项**:
- [ ] SQL注入测试
- [ ] XSS跨站脚本
- [ ] CSRF防护
- [ ] 密码强度
- [ ] Session安全
- [ ] API鉴权

**工具**: OWASP ZAP
