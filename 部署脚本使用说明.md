## 📋 脚本详细说明

### 1. deploy.sh - 一键部署脚本

**功能**: 完整自动化部署，从系统初始化到服务启动

**用法**:
```bash
sudo bash deploy.sh [选项]

# 基本用法
sudo bash deploy.sh --domain www.pofeclife.top --email admin@pofeclife.top

# 开发模式
sudo bash deploy.sh --dev-mode --domain dev.pofeclife.top

# 非交互模式（适用于自动化）
sudo bash deploy.sh --non-interactive \
  --mysql-root-password YourPassword123 \
  --db-password YourDbPassword456 \
  --domain www.pofeclife.top
```

**参数说明**:
- `--domain`: 主域名
- `--email`: 管理员邮箱
- `--mysql-root-password`: MySQL root密码
- `--db-password`: 应用数据库密码
- `--dev-mode`: 开发模式部署
- `--skip-ssl`: 跳过SSL证书配置
- `--non-interactive`: 非交互模式

### 2. manage_system.sh - 系统管理脚本

**功能**: 系统启动、停止、重启、监控

**用法**:
```bash
sudo bash manage_system.sh <命令>

# 基本命令
sudo bash manage_system.sh start      # 启动整个系统
sudo bash manage_system.sh stop       # 停止整个系统
sudo bash manage_system.sh restart    # 重启整个系统
sudo bash manage_system.sh status     # 查看系统状态
sudo bash manage_system.sh health     # 健康检查

# 日志查看
sudo bash manage_system.sh logs       # 查看日志
sudo bash manage_system.sh logs -f    # 实时查看日志

# 服务管理
sudo bash manage_system.sh start-app  # 仅启动应用
sudo bash manage_system.sh restart-web # 仅重启Web服务

# 维护命令
sudo bash manage_system.sh backup     # 备份数据库
sudo bash manage_system.sh clean      # 清理系统
sudo bash manage_system.sh monitor    # 实时监控
```

## 🆘 常见问题

### 1. 部署失败

**症状**: 脚本执行过程中出现错误

**解决方案**:
```bash
# 查看详细错误信息
sudo bash deploy.sh 2>&1 | tee deploy.log

# 检查系统要求
cat /etc/os-release
free -h
df -h

# 重新运行脚本
sudo bash deploy.sh --skip-system-init  # 跳过已完成步骤
```

### 2. 服务无法启动

**症状**: 部署成功但服务无法访问

**解决方案**:
```bash
# 检查服务状态
sudo bash manage_system.sh status

# 查看错误日志
sudo journalctl -u lsalumni -f
sudo tail -f /var/log/nginx/lsalumni_error.log

# 重启服务
sudo bash manage_system.sh restart
```

### 3. SSL证书问题

**症状**: HTTPS无法访问或证书警告

**解决方案**:
```bash
# 检查证书状态
sudo certbot certificates

# 重新申请证书
sudo certbot --nginx -d www.pofeclife.top --force-renewal

# 重启Nginx
sudo systemctl restart nginx
```

### 4. 数据库连接问题

**症状**: 应用无法连接数据库

**解决方案**:
```bash
# 检查MySQL服务
sudo systemctl status mysql

# 测试数据库连接
mysql -u lsalumni -p lsalumni

# 重置密码
mysql -u root -p -e "ALTER USER 'lsalumni'@'localhost' IDENTIFIED BY 'new_password';"
```
