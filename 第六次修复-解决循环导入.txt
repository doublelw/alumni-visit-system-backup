==========================================
问题诊断
==========================================

**循环导入错误**

错误日志：
```
ImportError: cannot import name 'db' from partially initialized module 'app'
(most likely due to a circular import) (/app/app.py)
```

**导入循环链：**
```
app.py (根目录)
  ↓ 调用 create_app()
backend/app/__init__.py
  ↓ 导入 from .routes.auth import auth_bp
backend/app/routes/auth.py
  ↓ from app import db (❌ 错误：指向根目录app.py)
app.py (根目录) ← 循环！
```

**根本原因：**
- 根目录有 app.py（入口文件）
- Flask应用包在 backend/app/
- 29个路由文件使用 `from app import db`
- Python将 `app` 解析为根目录的 app.py 模块
- 但 db 实际定义在 backend/app/__init__.py 中

==========================================
修复方案
==========================================

### 方案选择

**方案A（已采用）：重命名根目录 app.py**
- 改为 main.py
- 修改 Dockerfile 启动命令
- 优点：彻底解决命名冲突，只需修改2个文件
- 缺点：无

**方案B（未采用）：修改29个路由文件**
- 将所有 `from app import db` 改为相对导入
- 优点：保留 app.py 名称
- 缺点：需要修改29个文件，工作量大

### 具体修改

#### 1. 重命名 app.py → main.py
```bash
mv app.py main.py
```

#### 2. 修改 Dockerfile 启动命令
```diff
- CMD ["sh", "-c", "gunicorn app:application --bind 0.0.0.0:${PORT:-5000} ..."]
+ CMD ["sh", "-c", "gunicorn main:app --bind 0.0.0.0:${PORT:-5000} ..."]
```

#### 3. 更新部署脚本
- deployment_scripts/deploy_to_welife.py
- 将 app.py 改为 main.py

==========================================
技术说明
==========================================

### Python模块导入机制

**问题原理：**

当Python执行 `from app import db` 时：
1. 首先在 sys.path 中查找名为 `app` 的模块
2. 如果根目录有 app.py，将其作为 `app` 模块
3. 尝试从该模块中导入 `db` 属性
4. 但 `db` 实际在 backend/app/__init__.py 中

**解决方案原理：**

将 app.py 重命名为 main.py：
1. Python不再将 `app` 解析为根目录模块
2. `from app import db` 现在正确指向 backend/app 包
3. 循环导入被打破

### 为什么这个方案更好？

**对比其他方案：**

| 方案 | 修改文件数 | 风险 | 可维护性 |
|------|-----------|------|----------|
| 重命名app.py | 2个 | 低 | 高 |
| 修改29个路由文件 | 29个 | 中 | 中 |
| 改用绝对导入 | 30+个 | 高 | 低 |

**优势：**
- 一次性解决命名冲突
- 避免修改大量代码
- 更符合Python命名规范
- main.py 是常见的入口文件名
