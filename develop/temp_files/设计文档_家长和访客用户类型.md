## 二、家长（parent）设计

### 1. 使用场景
- 家长为在校学生申请入校（家长会、看望孩子、接送等）
- 家长可为自己申请，也可为孩子申请
- 需要验证与学生的关系

### 2. 家长与学生的关联方式

#### 数据库关联（已实现）
User模型中已有字段：
```python
# 家长-学生关系
parent_student_id = db.Column(db.Integer, db.ForeignKey('users.id'), nullable=True, comment='家长关联的学生ID')
student_parent_id = db.Column(db.Integer, db.ForeignKey('users.id'), nullable=True, comment='学生关联的家长ID')

# 关系定义
parent_students = db.relationship('User', foreign_keys=[parent_student_id], ...)
student_parents = db.relationship('User', foreign_keys=[student_parent_id], ...)
```

#### 关联逻辑
**方式1：家长账号关联多个学生**
```python
# 家长账号
parent_user = User(
    username='P2024001',
    real_name='张三父亲',
    user_type='parent',
    phone='13800138000',
    # 家长不直接关联学生，通过验证码或申请记录关联
)

# 关联通过VisitApplication实现

    applicant_id=parent_user.id,  # 家长申请
    target_student_id=student_user.id,  # 要看望的学生
    relationship='父女',  # 关系描述

```

**方式2：学生账号记录家长信息**
```python

    username='S2024001',
    real_name='张三',
    user_type='student',
    student_parent_id=parent_user.id  # 关联家长
)
```

### 3. 家长注册流程

#### 方案A：独立注册（推荐）
```
1. 家长注册账号（user_type='parent'）
2. 填写基本信息：姓名、手机、学生姓名、关系
3. 系统管理员审核：
   - 验证家长信息
   - 关联学生账号（通过学号匹配）
   - 审核通过后家长可申请入校
```

#### 方案B：由教师/管理员添加
```
1. 教师在管理后台添加家长
2. 选择关联的学生
3. 设置家长信息
4. 生成家长账号密码
```

## 三、访客（visitor）设计

### 1. 使用场景
- 临时访客（维修、送货、商务洽谈等）
- 短期访问（1-3天）
- 不需要长期账号

### 2. 访客账号特点
- **临时性**: 访客账号可设置过期时间
- **限制性**: 只能访问特定区域
- **审核严格**: 需要被访人邀请或审批

### 3. 访客创建方式

#### 方案A：被访人创建访客邀请
```
1. 教师/学生/校友创建"访客邀请"
2. 填写访客信息：姓名、手机、身份证号、访问时间、访问目的
3. 生成临时访问码
4. 访客使用访问码+验证码入校
```

#### 方案B：访客自行注册
```
1. 访客注册账号（user_type='visitor'）
2. 填写访问信息：被访人、访问时间、访问目的
3. 被访人确认或管理员审批
4. 审核通过后生成临时通行证
```

### 4. 访客数据库字段扩展
```python
class User(db.Model):
    # ... 现有字段 ...

    # 访客专属字段
    visitor_expiry_date = db.Column(db.DateTime, nullable=True, comment='访客账号过期时间')
    visitor_inviter_id = db.Column(db.Integer, db.ForeignKey('users.id'), nullable=True, comment='邀请人ID')
    visitor_visit_purpose = db.Column(db.String(200), nullable=True, comment='访问目的')
    visitor_allowed_areas = db.Column(db.Text, nullable=True, comment='允许访问的区域（JSON）')
```

### 5. 访客权限设计

### 3. 后端API更新

#### 用户创建API
```python
@auth_bp.route('/register', methods=['POST'])
def register():
    data = request.get_json()
    user_type = data.get('user_type')

    # 家长注册
    if user_type == 'parent':
        student = User.query.filter_by(student_id=data.get('parent_student_id')).first()
        if not student:
            return jsonify({'error': '未找到关联的学生'}), 400
        data['student_parent_id'] = student.id

    # 访客注册
    elif user_type == 'visitor':
        # 设置过期时间（默认3天）
        from datetime import timedelta
        expiry_days = int(data.get('visitor_expiry_days', 3))
        data['visitor_expiry_date'] = datetime.now() + timedelta(days=expiry_days)

        # 处理访问区域
        allowed_areas = data.get('allowed_areas', [])
        data['visitor_allowed_areas'] = json.dumps(allowed_areas)

    # 创建用户
    user = User(**data)
    db.session.add(user)
    db.session.commit()

    return jsonify({'success': True, 'user_id': user.id})
```

## 五、推荐方案总结

### 家长（parent）
- **user_type**: `'parent'`
- **注册方式**: 独立注册或教师添加
- **关联方式**: 通过`parent_student_id`字段关联学生
- **权限**: 可为孩子申请入校、查看孩子信息

### 访客（visitor）
- **user_type**: `'visitor'`
- **创建方式**: 被访人邀请或自行注册
- **时效性**: 设置`visitor_expiry_date`过期时间
- **权限**: 受限访问、限时有效

### 关键字段
```python
# 家长字段
parent_student_id = db.Column(db.Integer, db.ForeignKey('users.id'))  # 家长→学生

visitor_visit_purpose = db.Column(db.String(200))         # 访问目的
```
