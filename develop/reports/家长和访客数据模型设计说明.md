# 家长和访客数据模型设计说明

## 📋 设计概述

本系统基于 **User表** 作为用户核心数据表，通过 **user_type** 字段区分不同用户类型：
- `alumni` - 校友
- `teacher` - 教师
- `student` - 学生
- `parent` - 家长
- `visitor` - 访客

## 🗂️ 数据模型结构

### 3. ParentStudentRelation表（家长-学生关联中间表）

**文件**: `backend/app/models/parent_profile.py`

```python
class ParentStudentRelation:
    id                      # 主键
    parent_id               # 家长User ID
    student_id              # 学生User ID

    # 关系属性
    relationship_type       # 关系类型
    is_primary              # 是否为主要监护人
    is_emergency_contact    # 是否为紧急联系人
    can_pick_up             # 是否可以接学生离校
    priority                # 优先级（数字越小优先级越高）

    # 审批状态
    is_verified             # 亲子关系是否已验证
    verified_at             # 验证时间
    verified_by             # 验证人（班主任/管理员）

    # 时间戳
    created_at
    updated_at

    # 唯一约束
    UniqueConstraint(parent_id, student_id)  # 一个家长对一个学生只能有一条记录
```

**设计优势**：
- ✅ 支持一对多：一个家长可以有多个孩子
- ✅ 支持多对一：一个孩子可以有多个家长（父母、祖父母等）
- ✅ 记录关系属性：是否主要监护人、是否可接孩子等
- ✅ 支持关系验证：班主任可验证亲子关系

## 🔗 家长与学生关联的三种方式

### 方式1：使用ParentStudentRelation中间表（推荐）✅

**优势**：
- 支持一对多、多对多关系
- 可以记录多个家长（父亲、母亲、祖父母等）
- 可以记录关系属性（主要监护人、是否可接孩子等）
- 支持关系验证

**使用场景**：
```python
# 创建家长-学生关联
relation = ParentStudentRelation(
    parent_id=2,           # 张父的User ID
    student_id=1,          # 张三的User ID
    relationship_type='father',
    is_primary=True,
    can_pick_up=True,
    priority=1
)
db.session.add(relation)
db.session.commit()

# 查询学生的所有家长
parents = ParentStudentRelation.query.filter_by(student_id=1).all()

# 查询家长的所有孩子
children = ParentStudentRelation.query.filter_by(parent_id=2).all()
```

### 方式2：使用User表的关联字段（已存在）

**优势**：
- 简单直接
- 不需要额外的中间表

**劣势**：
- 一对一关系（一个学生只能关联一个家长）
- 不够灵活

**使用场景**：
```python
# 在User表中直接关联（已存在的字段）
student = User.query.get(1)
student.parent_student_id = 2  # 关联父亲
db.session.commit()
```

### 方式3：组合使用（最灵活）✅✅✅

```python
# User表的字段用于快速查询主要联系人
student.parent_student_id = 2  # 主要家长（父亲）

# ParentStudentRelation表记录所有家长关系
relation1 = ParentStudentRelation(
    parent_id=2,
    student_id=1,
    relationship_type='father',
    is_primary=True
)
relation2 = ParentStudentRelation(
    parent_id=3,
    student_id=1,
    relationship_type='mother',
    is_primary=False
)
db.session.add_all([relation1, relation2])
db.session.commit()
```

## 🎯 设计总结

### 核心设计原则
1. **User表统一管理**：所有用户类型共用User表，通过user_type区分
2. **Profile表扩展信息**：每种用户类型有专属的Profile表存储详细信息
3. **中间表管理复杂关系**：家长-学生多对多关系使用中间表管理
4. **灵活可扩展**：易于添加新的用户类型和关系

### 关键优势
- ✅ **统一认证**：所有用户使用同一套登录系统
- ✅ **灵活关联**：支持一个学生多个家长，一个家长多个孩子
- ✅ **权限管理**：通过关系属性控制权限（是否可接孩子、是否主要监护人等）
- ✅ **访客管理**：支持访客申请、审批、访问记录、黑名单等全流程
- ✅ **微信集成**：支持微信H5登录
