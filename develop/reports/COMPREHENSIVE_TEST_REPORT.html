🔧 第三部分：技术实现 • Part 3: Technical Implementation

HMAC码生成算法 • HMAC Generation

校友/家长/访客/活动报名： HMAC-SHA256(phone + wechat_password + timestamp) 有效时间窗口：3分钟（访客24小时） 学生离校： HMAC-SHA256(student_id + 'STUDENT_EXIT' + today_date_timestamp) 有效时间窗口：当天23:59前

外网/内网分离架构 • Network Separation

外网数据库 (external_network.db)： • access_codes表：16条记录 • visitor_cache表：16条记录 • 仅存储：访问码 + 手机号（最小数据原则） 内网数据库 (alumni_system_dev.db)： • users表：完整用户信息（27个用户） • visit_applications表：访问申请记录 • 完整信息存储，包含所有个人详细信息 数据流： 单向迁移（外网 → 内网） 外网仅提供验证接口，内网存储完整数据

API端点结构 • API Endpoints

管理端API (Admin APIs)： • POST /api/auth/login - 管理员登录 • GET /api/admin/users - 获取用户列表（支持分页） • POST /api/admin/users - 添加用户 • GET /api/organization/list - 获取组织列表 • POST /api/organization - 创建组织 • GET /api/visits/applications - 获取访问申请 • GET /api/visits/records - 获取访问记录 访客端API (Visitor APIs)： • POST /api/guard/verify - 门卫验证（支持5种类型） • POST /api/visitor/register - 访客登记 • POST /api/external-visitor/generate-code - 生成访客码 • POST /api/event/register - 活动报名

时间窗口配置 • Time Window Configuration

门卫验证时间窗口： • 访客类型：24小时（1440分钟） • 校友/家长/活动：3分钟 • 学生离校：当天23:59:59前有效 教师审批时间窗口： • 访客申请：24小时内审批有效
