# 校友入校电子卡系统需求规格说明书

## 1. 引言

### 1.1 编写目的
本文档旨在详细描述校友入校电子卡系统的功能需求、非功能需求以及系统设计约束，为系统开发、测试和维护提供依据。

### 11.2 项目背景
随着高校校友网络的不断扩大，校友返校访问的需求日益增加。为了规范校友入校管理流程，提高校园安全管理水平，特开发此校友入校电子卡系统。

### 1.3 读者对象
- 项目经理
- 系统架构师
- 开发人员
- 测试人员
- 系统管理员
- 最终用户

### 1.4 术语定义
| 术语 | 定义 |
|------|------|
| 校友 | 从本校毕业的人员 |
| 接待人 | 负责接待校友的在校教师或工作人员 |
| 访问申请 | 校友提交的入校访问请求 |
| 人脸识别 | 基于人脸特征进行身份验证的技术 |
| 批量授权 | 管理员或教师对特定群体进行统一访问授权 |

## 2. 项目概述

### 2.1 产品描述
校友入校电子卡系统是一个集校友信息管理、访问申请审批、人脸识别验证、车辆管理、数据统计分析于一体的综合性校园管理系统。

### 2.2 产品功能
- 校友注册与信息管理
- 访问申请与审批
- 人脸识别入校验证
- 车辆登记与管理
- 权限管理与角色控制
- 数据统计与分析
- 微信公众号集成

### 2.3 用户特点
| 用户角色 | 描述 | 主要需求 |
|---------|------|---------|
| 校友 | 系统的主要使用者 | 便捷的注册流程、访问申请、人脸识别入校 |
| 接待人 | 校友的联系教师 | 审核校友信息、确认访问申请 |
| 保安 | 校园安全管理人员 | 验证入校人员身份、记录进出信息 |
| 管理员 | 系统管理人员 | 用户管理、权限分配、数据统计 |

### 2.4 运行环境
- 服务器端：Python 3.7+，Flask框架，MySQL数据库
- 客户端：现代浏览器（Chrome、Firefox、Safari等）
- 移动端：微信公众号
- 硬件设备：人脸识别终端、闸机设备

## 3. 功能需求

### 3.1 用户管理模块

#### 3.1.1 校友注册
**功能描述：** 校友通过系统注册个人信息，完成身份认证。
**功能流程：**
1. 校友访问注册页面
2. 填写基本信息（用户名、密码、真实姓名、邮箱、手机号）
3. 填写校友档案信息（学号、毕业年份、班级、院系、专业、身份证号）
4. 填写联系方式（接待人信息、紧急联系人）
5. 提交注册申请
6. 系统发送审核通知给接待人
7. 接待人审核通过后，校友获得访问权限

**输入：**
- 用户名、密码
- 真实姓名、邮箱、手机号
- 学号、毕业年份、班级、院系、专业
- 身份证号
- 接待人姓名及联系方式
- 紧急联系人信息

**输出：**
- 注册成功提示
- 审核状态通知

**异常处理：**
- 重复用户名/邮箱提示
- 信息格式验证失败提示
- 网络异常处理

#### 3.1.2 用户登录
**功能描述：** 已注册用户通过用户名和密码登录系统。
**功能流程：**
1. 用户输入用户名和密码
2. 系统验证用户信息
3. 验证通过后生成访问令牌
4. 跳转到用户主页

**输入：**
- 用户名
- 密码

**输出：**
- 登录成功跳转
- 登录失败提示

### 3.2 访问管理模块

#### 3.2.1 访问申请
**功能描述：** 校友提交入校访问申请，需要提前一小时预约。
**功能流程：**
1. 校友登录系统
2. 进入访问申请页面
3. 填写访问信息（访问日期、时间、目的、拜访对象）
4. 提交申请
5. 系统通知接待人审核
6. 审核通过后生成访问凭证

**输入：**
- 访问日期
- 访问时间段
- 访问目的
- 拜访对象/部门

**输出：**
- 申请提交成功提示
- 审核结果通知

**约束条件：**
- 需提前1小时申请
- 访问时间不能超出规定范围

#### 3.2.2 批量授权
**功能描述：** 管理员或教师可以一次授权多人访问。
**功能流程：**
1. 管理员/教师登录系统
2. 进入批量授权页面
3. 选择授权类型（按班级、年级、院系）
4. 选择目标群体
5. 设置访问时间范围和权限级别
6. 提交授权申请

**输入：**
- 授权类型
- 目标群体
- 访问时间范围
- 权限级别

**输出：**
- 授权成功提示
- 授权记录

### 3.3 人脸识别模块

#### 3.3.1 人脸注册
**功能描述：** 校友注册人脸信息用于入校验证。
**功能流程：**
1. 校友登录系统
2. 进入人脸注册页面
3. 上传人脸照片或通过摄像头采集
4. 系统处理人脸信息并保存特征数据
5. 注册成功提示

**输入：**
- 人脸照片或摄像头采集图像

**输出：**
- 注册成功/失败提示

#### 3.3.2 人脸识别验证
**功能描述：** 校友入校时通过人脸识别验证身份。
**功能流程：**
1. 校友到达校门闸机
2. 面对人脸识别设备
3. 系统采集人脸图像
4. 与注册的人脸特征进行比对
5. 验证通过则开启闸机
6. 记录进出时间

**输入：**
- 实时采集的人脸图像

**输出：**
- 验证结果
- 闸机控制信号
- 进出记录

### 3.4 车辆管理模块

#### 3.4.1 车辆登记
**功能描述：** 校友登记车辆信息，便于车辆进出校园管理。
**功能流程：**
1. 校友登录系统
2. 进入车辆登记页面
3. 填写车辆信息（车牌号、车型、颜色等）
4. 提交登记申请
5. 审核通过后车辆获得进出权限

**输入：**
- 车牌号
- 车型
- 颜色
- 车主信息

**输出：**
- 登记成功/失败提示
- 审核状态

#### 3.4.2 车辆进出记录
**功能描述：** 记录校友车辆进出校园的时间和信息。
**功能流程：**
1. 车辆到达校门识别设备
2. 系统识别车牌号
3. 验证车辆登记信息
4. 记录进出时间
5. 开启道闸

### 3.5 权限管理模块

#### 3.5.1 角色权限控制
**功能描述：** 根据用户角色分配不同的系统权限。
**角色权限：**
- 校友：个人信息管理、访问申请、人脸注册、车辆登记
- 接待人：审核校友信息、确认访问申请
- 保安：查看访问信息、管理现场进出
- 管理员：系统配置、用户管理、权限分配、数据统计

#### 3.5.2 访问级别管理
**功能描述：** 设置不同用户的访问级别和时间限制。
**功能流程：**
1. 管理员登录系统
2. 进入权限管理页面
3. 选择用户或用户组
4. 设置访问级别和时间限制
5. 保存设置

### 3.6 统计分析模块

#### 3.6.1 人员统计
**功能描述：** 按人员维度统计访问情况。
**统计内容：**
- 各校友访问频次
- 活跃校友排名
- 访问目的分布

#### 3.6.2 年级统计
**功能描述：** 按年级维度统计访问情况。
**统计内容：**
- 各年级校友访问情况
- 年级访问趋势分析

#### 3.6.3 班级统计
**功能描述：** 按班级维度统计访问情况。
**统计内容：**
- 各班级校友访问情况
- 班级活跃度分析

## 4. 非功能需求

### 4.1 性能需求
- 系统响应时间：页面加载时间不超过3秒
- 并发用户数：支持至少1000个并发用户
- 人脸识别响应时间：单次识别不超过2秒

### 4.2 安全需求
- 用户密码加密存储
- 敏感数据传输加密
- 人脸特征数据加密存储
- 访问日志记录完整

### 4.3 可用性需求
- 系统可用性：99.5%以上
- 支持主流浏览器
- 提供友好的用户界面

### 4.4 兼容性需求
- 支持Windows、Linux、macOS操作系统
- 支持Chrome、Firefox、Safari等主流浏览器
- 支持微信公众号访问

## 7. 部署方案

### 7.1 硬件环境
- 服务器：至少4核CPU，8GB内存，100GB硬盘空间
- 人脸识别终端：支持人脸识别的闸机设备
- 网络设备：千兆交换机，路由器

### 7.2 软件环境
- 操作系统：Linux Ubuntu 18.04+
- Web服务器：Nginx
- 应用服务器：Gunicorn
- 数据库：MySQL 5.7+
- Python版本：3.7+

### 7.3 部署步骤
1. 安装依赖软件
2. 配置数据库
3. 部署应用代码
4. 配置Web服务器
5. 启动服务
6. 验证部署结果
